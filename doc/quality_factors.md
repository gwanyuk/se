시스템 명칭: 클라우드 파일 공유 시스템 (미니 드라이브)



대상 시스템 품질 요소 측정



(1) 사용자가 원하는 기대치는 무엇인가?

사용성 (Usability) : 특별한 프로그램 설치 없이 웹 브라우저를 통해 직관적이고 간단하게 파일을 업로드하고 다운로드할 수 있어야 합니다.

성능 (Performance) :  필요한 파일을 빠르게 찾을 수 있도록 검색 기능과 파일 다운로드 기능이 지연 없이 신속하게 동작해야 합니다.

신뢰성 (Reliability) : 실수로 파일을 삭제하거나 덮어쓰더라도 이전 버전으로 복원하거나 삭제된 파일을 일정 기간 내에 복구할 수 있어 데이터 유실을 방지해야 합니다.

보안성 (Security - User View) : 파일 공유 시 조회, 수정, 댓글 등의 권한을 세분화하여 설정하고, 외부 링크 공유 시 만료 기간을 두어 안전하게 협업할 수 있어야 합니다.





(2) 개발자 혹은 유지보수 담당자가 원하는 기대치는 무엇인가?

확장성 및 유지보수성 (Extensibility \& Maintainability) : 향후 문서 공동 편집, 자동 파일 분류, AI 기반 검색 기능 등이 추가될 수 있으므로, 새로운 모듈을 쉽게 통합할 수 있는 유연한 아키텍처로 설계되어야 합니다.

효율성 (Efficiency) : 약 200명 규모의 직원이 동시에 시스템을 사용하더라도 병목 현상 없이 안정적으로 서버 리소스가 관리되고 동작해야 합니다.

테스트 용이성 (Testability) : 파일 저장, 폴더 구성, 검색, 권한 제어 등 주요 기능이 독립적으로 구현되어 빠르고 정확하게 단위 테스트 및 통합 테스트를 수행할 수 있어야 합니다.





(3) 개발 조직의 관리자가 원하는 기대치는 무엇인가?

보안성 및 통제성 (Security \& Control) : 조직의 중요한 데이터가 저장되므로 데이터 전송 시 암호화를 적용하고, 외부 유출 위험을 차단할 수 있는 철저한 접근 권한 관리(RBAC) 시스템이 갖춰져야 합니다.

관리 용이성 (Manageability) : 사용자 계정 생성 및 삭제, 직원별 저장 공간 할당 및 그룹 관리 기능을 통해 시스템을 중앙에서 효율적으로 통제할 수 있어야 합니다.

업무 효율성 (Business Value) : 최신 버전 관리의 어려움과 파일 검색에 소요되는 시간 낭비 문제를 근본적으로 해결하여 조직 전체의 프로젝트 협업 생산성을 높이는 것이 궁극적인 목표입니다.
