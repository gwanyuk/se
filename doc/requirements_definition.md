## 1. 개요(Introduction)
### 1.1 문서의 목적
이 문서는 클라우드 기반 파일 공유 시스템인 'Mini Drive' 프로젝트 개발을 위한 요구사항을 명확히 식별하고 정의하는 데 목적이 있다. 기능적/비기능적 요구사항 및 인터페이스 요구사항을 명시하여 설계와 구현의 기준을 제공한다.

### 1.2 프로젝트 개요
#### 1.2.1 프로젝트 정의
Mini Drive는 사용자가 개인 파일을 클라우드에 업로드 및 다운로드하고, 필요한 파일을 다른 사용자와 공유할 수 있도록 지원하는 시스템이다. 파일 관리의 편의성을 높이고 구성원 간의 원활한 데이터 공유 환경을 제공하는 것을 목표로 한다.[cite: 1]

#### 1.2.2 주요 기능 설명
| ID | 설명 |
| :--- | :--- |
| F01 | 사용자 계정 관리 및 인증 (회원가입, 로그인)[cite: 1] |
| F02 | 클라우드 저장소로 파일 업로드 및 다운로드[cite: 1] |
| F03 | 다른 사용자에게 파일 공유 링크 생성 및 제공[cite: 1] |
| F04 | 디렉토리(폴더) 생성, 수정, 삭제 등 파일 계층 관리[cite: 1] |

### 1.3 관련 문서, 용어 및 약어
- **클라우드 스토리지 (Cloud Storage)**: 인터넷을 통해 데이터를 원격 서버에 저장하고 관리하는 서비스.
- **동기화 (Synchronization)**: 여러 디바이스에서 동일한 최신 상태의 파일 트리를 유지하는 과정.
- **API**: 응용 프로그램에서 사용할 수 있도록 운영체제나 프로그래밍 언어가 제공하는 기능을 제어할 수 있게 만든 인터페이스.

## 2. 요구사항
### 2.1 기능적 요구사항
| 분류 | 요구사항 | 우선순위 |
| :--- | :--- | :--- |
| FR-001 | 사용자는 아이디, 비밀번호, 이메일을 입력하여 시스템에 회원가입 및 로그인을 할 수 있다. | 상 |
| FR-002 | 사용자는 자신의 로컬 환경에서 클라우드 저장소로 파일을 업로드할 수 있다. | 상 |
| FR-003 | 사용자는 클라우드에 저장된 파일을 선택하여 로컬 환경으로 다운로드할 수 있다. | 상 |
| FR-004 | 사용자는 생성된 특정 공유 링크를 통해 타인에게 파일을 공유할 수 있다. | 중 |
| FR-005 | 사용자는 클라우드 내에 디렉토리(폴더)를 생성, 수정, 삭제하여 파일을 계층적으로 관리할 수 있다. | 상 |

### 2.2 비기능적 요구사항
| 분류 | 요구사항 | 우선순위 |
| :--- | :--- | :--- |
| NFR-001 | 시스템은 다수의 동시 접속 환경에서도 안정적인 파일 전송 속도를 유지해야 한다. | 중 |
| NFR-002 | 모든 사용자의 업로드 및 다운로드 요청에 대해 시스템은 3초 이내로 응답해야 한다. | 상 |
| NFR-003 | 시스템은 사용자의 비밀번호 및 주요 개인정보를 안전한 해시 알고리즘으로 암호화하여 저장한다. | 상 |

### 2.3 인터페이스 요구사항
| 분류 | 요구사항 | 우선순위 |
| :--- | :--- | :--- |
| IR-001 | 시스템은 처음 사용하는 사람도 쉽게 인지할 수 있는 직관적인 UI를 제공해야 한다. | 상 |
| IR-002 | 클라이언트와 서버 간의 데이터 통신은 RESTful API 규격을 준수하여 이루어져야 한다. | 상 |

## 3. 품질 요구사항
### 3.1 신뢰성 (Reliability)
- 대용량 파일 전송 중 네트워크 연결 불안정으로 인한 데이터 손실이 발생하지 않아야 하며 무결성을 보장해야 한다.

### 3.2 사용성 (Usability)
- 별도의 학습이나 매뉴얼 없이도 파일 관리에 필요한 핵심 기능에 바로 접근할 수 있도록 동선이 설계되어야 한다.

### 3.3 보안성 (Security)
- 권한이 없는 사용자의 접근을 철저히 차단하며, 본인 소유의 파일과 명시적으로 공유된 파일에만 접근 권한을 부여한다.

## 5. 기타 고려 사항
- 3인으로 구성된 프로젝트 팀의 원활한 병렬 개발을 위해 초기 디렉토리 구조 정의 및 역할 분담을 명확히 한다.
- `C:\Dev\OOS_Project` 등 팀원별 로컬 개발 디렉토리 경로가 상이할 수 있으므로, 환경 변수와 상대 경로를 적극 활용하여 빌드 및 동기화 충돌을 방지한다.
